# Звіт про порівняння алгоритмів пошуку підрядка

У цьому дослідженні ми порівнювали продуктивність трьох відомих алгоритмів пошуку підрядка - Кнута-Морріса-Пратта (КМП), Боєра-Мура та Рабіна-Карпа - на двох текстах різної довжини та складності. Метою було визначити, який алгоритм є найефективнішим для кожного з текстів з урахуванням реальних і вигаданих патернів.

Для кожного алгоритму ми вимірювали час виконання пошуку заданого підрядка в тексті. Тестування проводилося за допомогою функції `measure_algorithm_time`, яка використовувала `timeit` для заміру середнього часу виконання алгоритму на основі 10 прогонів.

## Час виконання для реального підрядка:

### Стаття 1:

- **KMP**: 0.0030 сек
- **Boyer-Moore**: 0.0011 сек
- **Rabin-Karp**: 0.0070 сек

### Стаття 2:

- **KMP**: 0.0021 сек
- **Boyer-Moore**: 0.0007 сек
- **Rabin-Karp**: 0.0049 сек

## Час виконання для вигаданого підрядка:

### Стаття 1:

- **KMP**: 0.0036 сек
- **Boyer-Moore**: 0.0009 сек
- **Rabin-Karp**: 0.0080 сек

### Стаття 2:

- **KMP**: 0.0024 сек
- **Boyer-Moore**: 0.0006 сек
- **Rabin-Karp**: 0.0054 сек

## Висновки:

Алгоритм **Боєра-Мура** є найшвидшим для обох статей та для обох типів підрядків (реальний та вигаданий). Це свідчить про високу ефективність алгоритму Боєра-Мура в даних умовах пошуку.

Алгоритм **Рабіна-Карпа** показує найповільніше виконання серед трьох розглянутих алгоритмів. Обчислення хеш-функцій навантажує систему і ймовірно, є причиною його повільнішого виконання порівняно з іншими алгоритмами.

Алгоритм **КМП** демонструє середні результати, швидше за Рабіна-Карпа, але повільніше за Боєра-Мура. 

### Загальна рекомендація:

Алгоритм **Боєра-Мура** є оптимальним вибором для нашого варіанту пошуку підрядків у тексті. Для інших задач потрібно проводити додаткові заміри.
